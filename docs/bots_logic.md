# 🧠 Логика ИИ (bots_logic.md)

Документация описывает алгоритмы, систему приоритетов и эволюцию принятия решений для трех типов ботов, реализованных в проекте.

---

## 1. PrimitiveBot (Базовый уровень)
Предназначен для разминки игрока и тестирования механик захвата.

* **Алгоритм:** Случайный выбор с приоритетом взятия.
* **Приоритеты:**
    1.  Собирает все легальные ходы (для шахмат учитывается проверка на шах).
    2.  Если есть возможность взять фигуру (`isCapture`), выбирает случайное взятие.
    3.  Если взятий нет, совершает случайное легальное перемещение.
* **Особенность:** Не обладает "памятью" или прогнозированием.

---

## 2. SmartCheckersBot (Жадный эвристик)
Бот для игры за шашки, использующий взвешенную оценку текущего состояния (без поиска вглубь).

* **Алгоритм:** Оценочная функция одного хода (`evaluateMove`).
* **Критерии оценки:**
    * **Взятие:** Бонус +1000 за каждую съеденную фигуру (стимул к Multi-jump) + бонус на основе `penalty` шахматной фигуры.
    * **Безопасность:** Жесткий штраф (-1500), если клетка назначения простреливается шахматами.
    * **Защита базы:** Штраф за уход с последней линии (защита от превращения пешек).
    * **Поддержка:** Бонус за наличие союзной шашки сзади по диагонали.
    * **Позиция:** Предпочтение центра доски и избегание бортов.

---

## 3. SmartChessBot (Minimax Гроссмейстер)
Основной противник, использующий дерево предсказаний и глубокий анализ.

### Алгоритм: Minimax + Alpha-Beta Pruning
Бот строит дерево ходов на глубину **4 полухода**. Для оптимизации используется Alpha-Beta отсечение, которое игнорирует заведомо проигрышные ветки, ускоряя расчет в десятки раз.

### Quiescence Search (Поиск покоя)
Для предотвращения «эффекта горизонта» (когда бот не видит взятие на 5-м ходу), используется добор. Если на предельной глубине возможны взятия, бот продолжает считать их (до глубины -6), пока позиция не стабилизируется.

### Оценочная функция (`evaluateBoard`)
Итоговый вес позиции рассчитывается как разница между материалом белых и черных с учетом позиционных факторов.

| Фигура / Статус | Вес (Points) | Описание |
| :--- | :--- | :--- |
| **Белый Король** | 10,000,000 | Жизненно важная цель. Потеря = `-LOSE_SCORE`. |
| **Ферзь** | 1,200,000 | Абсолютный приоритет сохранения. |
| **Ладья** | 600,000 | Высокая ценность. |
| **Конь / Слон** | 400,000 | Средняя ценность. |
| **Шашка (Черная)** | 45,000 | Ценность выше белой пешки. |
| **Белая Пешка** | 30,000 | Ценность растет при приближении к 0-му ряду. |

## 🛠️ Правила и Приоритеты (Decision Making)

1.  **Выживание Короля:** Попадание под шах оценивается как катастрофа (-20,000,000 к весу фигуры). Бот выйдет из-под шаха любой ценой.
2.  **Тотальное сохранение материала:** Веса шахматных фигур установлены так, что размен любой фигуры на обычную шашку считается невыгодным. Бот — не пацифист, но крайне дорожит своими силами.
3.  **DangerMap (Радар угроз):** Перед каждым ходом бот строит карту полей, которые могут атаковать шашки. Если фигура попадает в эту зону, её ценность в расчетах падает, что заставляет бота маневрировать.
4.  **Стимул к победе:** * Полное уничтожение черных или их блокировка (отсутствие ходов) = `WIN_SCORE`.
    * Появление у черных 2-х дамок = `LOSE_SCORE` (стимул не допускать прорыва).
5.  **Продвижение пешек:** Каждая клетка вперед для пешки дает бонус +2500, форсируя создание новых Ферзей.

* **v3:** Реализация Minimax и Alpha-Beta.
* **v4:** Внедрение Quiescence Search для борьбы с двойными прыжками шашек.
